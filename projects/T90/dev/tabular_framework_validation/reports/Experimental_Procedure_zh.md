## 8. 2026-04-02 - 关于工业标签模糊性的方向修正

- 背景：
  - 用户明确指出当前工业数据的标签并不是严格精确真值
  - 原因是 LIMS 人工测定存在天然误差，尤其在边界区附近更明显
  - 因此如果直接以 `8.45 ± 0.25` 做硬切分，强行生成刚性分类标签，后续模型极易学到边界噪声
- 方向结论：
  - 后续建模分析必须把“标签模糊性”作为第一约束，而不是默认标签绝对准确
  - 对当前项目，更合适的方向是：
    - 模糊的 `x`
    - 对应模糊的 `y`
    - 最终输出不合格风险概率，而不是单纯硬分类
- 对当前各分支的影响：
  - `centered_desirability` 分支保留
    - 因为它本质上已经不是硬标签，而是连续型软目标
  - `high_risk` 硬二分类分支降级为“对照支线”
    - 不再把它当成后续主线标签方案
  - 后续若继续做风险建模，应优先改为：
    - 软标签概率建模
    - 或模糊隶属度 / 风险度建模
- 记录口径：
  - 今后在本目录中若再出现硬标签分类实验，必须明确说明其作用仅限于：
    - baseline 对照
    - 或诊断性实验
  - 不应再将硬分类标签视为最终工业建模目标

  - unlike the centered_desirability branch, the soft target branch did not obtain a net gain from the tested weaker-compression representations
  - the current whole-window range-position reference remains the best validated main line for the soft target task
  - the result is now based on a controlled follow-up experiment rather than on a single negative segmented-window trial
